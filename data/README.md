\# Data Directory



\## Structure



\- `raw/` — Original, unmodified benchmark downloads (HuggingFace `datasets` format)

&#x20; - `truthfulqa/` — TruthfulQA (generation config), 817 examples

&#x20; - `halueval\_qa/` — HaluEval QA subset, 10,000 examples

&#x20; - `fever/` — FEVER via `pietrolesci/nli\_fever` reformatting, 208,346 examples



\- `processed/` — Unified-schema CSVs, one row per example

&#x20; - Schema: `id, source\_dataset, input\_text, reference\_answer, context, label`

&#x20; - `truthfulqa.csv`, `halueval.csv`, `fever.csv`



\- `benchmarks/` — Cleaned, deduplicated, split versions used for experiments

&#x20; - Each dataset split 80/10/10 into `\_train.csv`, `\_val.csv`, `\_test.csv`

&#x20; - Deduplicated on `input\_text`; rows with missing required fields dropped



\## Dataset Notes



\*\*TruthfulQA\*\*: Open-ended questions with a best/correct answer. No inherent

binary label — used for generation + downstream judgment (LLM answers the

question, detector flags if the answer diverges from `reference\_answer`).



\*\*HaluEval (QA)\*\*: Each example has a `right\_answer` and a `hallucinated\_answer`

generated by an LLM, plus `knowledge` (grounding context). Only the

right-answer side is currently in `processed/`/`benchmarks/`; hallucinated

answers will be used separately when building contrastive train pairs.



\*\*FEVER (via nli\_fever)\*\*: Claim (`hypothesis`) + evidence (`premise`) +

label. Label mapping: `0 = entailment`, `1 = neutral`, `2 = contradiction`.

Used primarily for the NLI-based detection method (`nli\_based\_detection.py`).

Deduplication on `input\_text` collapses claims that appeared against multiple

evidence sentences in the original data, reducing 208,346 → 87,836 unique

claims.
